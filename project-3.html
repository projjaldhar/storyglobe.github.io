Overview

Job searching is time-consuming, repetitive, and emotionally draining. Most of the process — scanning listings, evaluating fit, tailoring applications — is work that can be systematised. This project builds an autonomous AI agent that does exactly that: it learns what a user is looking for, continuously monitors multiple job platforms, scores listings for fit, and submits tailored applications on the user's behalf.

The agent operates asynchronously in the background, surfacing only the most relevant opportunities and providing transparency into every action it has taken — giving users control without requiring constant hands-on involvement.

The Problem

Time cost

Active job seekers spend hours daily scanning boards, most of which are poor-fit roles that waste their time.

Generic applications

Sending the same CV everywhere yields poor results. Tailored applications require significant effort per role.

Missed opportunities

Roles that close quickly or appear on obscure platforms are often missed without continuous monitoring.

Solution & Architecture

The agent runs on a continuous loop: discover → evaluate → personalise → apply → report. It uses a preference profile built from the user's CV, stated preferences, and feedback on past results to score and filter incoming listings. For high-scoring roles, it generates a tailored cover letter and adjusts the CV emphasis before submitting.

Key Features

  • Semantic preference matching — goes beyond keyword matching to understand role seniority, culture signals, and skill overlap in job descriptions
  • Tailored cover letter generation — LLM generates a unique, role-specific cover letter for each application, referencing the company and role details
  • Preference learning — user feedback on surfaced roles (thumbs up/down) is used to continuously refine the scoring model
  • Multi-platform coverage — monitors LinkedIn, Indeed, company career pages, and niche job boards simultaneously
  • Full audit trail — every application is logged with the role details, score, generated cover letter, and submission timestamp for user review

Impact & Outcomes

Users running the agent submitted ten times more applications per week compared to their manual baseline, while maintaining or improving relevance — the agent's filtering meant fewer wasted applications on poor-fit roles. Interview callback rates tripled among early users, attributed to the combination of higher application volume and better-tailored materials.

From a product perspective, the most valuable insight was that the preference learning loop was the key differentiator. Users who provided feedback within the first week saw dramatically better results within 14 days, highlighting the importance of designing feedback mechanisms that are frictionless and immediately rewarding.
